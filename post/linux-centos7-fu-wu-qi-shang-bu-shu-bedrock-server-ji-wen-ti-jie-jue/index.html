linux centos7服务器上部署bedrock-server及问题解决

2021-11-09 5 min read

###一. 源码下载：
目前我的官网已经提供bedrock-server的下载地址https://minecraft.net/zh-hans/download/server/bedrock/
1.直接下载到本地电脑，解压后将文件夹上传至linux服务器
2.在linux服务器上通过wget命令直接下载，免去了文件传输的麻烦
wget -c https://minecraft.azureedge.net/bin-linux/bedrock-server-1.7.0.13.zip
安装unzip
yum install -y unzip
解压
unzip -o -d /root/bedrock bedrock-server-1.7.0.13.zip
###二.运行
cd bedrock
LD_LIBRARY_PATH=. ./bedrock_server
如果你使用的是centos系统，这时候运行百分百会报错，因为这是已经编译好的程序压缩包，你使用的环境与他们所编译的环境是不同的，所以有一些动态链接库需要进行修改。
###三.解决汇总
1.

./bedrock_server: error while loading shared libraries: libssl.so.1.1: cannot open shared object file: No such file or directory

这个是个比较棘手的问题，因为它涉及到了openssl,如果一个操作失误就会导致服务器的网络连接崩溃，你再也无法通过远程连接到服务器，唯一的解决办法就是重装系统。
首先，下载：https://www.openssl.org/source/old/1.0.1/openssl-1.0.1e.tar.gz
执行./config shared zlib-dynamic 来生成Makefile文件。
cd openssl-1.0.1e
./config shared zlib-dynamic
通过make来生成生成libssl.so.1.0.0和libcrypto.so.1.0.0,这一步不要install，我们只需要编译好的文件，需要安装。
将这2个文件拷贝到 /usr/lib64/，然后执行以下2条生成软连接解决
cd /usr/lib64

ln -s libssl.so.1.0.0 libssl.so.10
ln -s libcrypto.so.1.0.0 libcrypto.so.10
到这一步，第一个问题已经基本解决了
2.

gcc command not found...

这是因为没有安装gcc，
yum -y install gcc
yum -y install gcc-c++
3.

c_zlib.c:25:19: fatal error: zlib.h: No such file or directory

在http://www.zlib.net/选择一个对应的版本下载，我下载的是 最新的1.2.11
下载完成后解压，cd到解压目录
构建静态库
./configure
make test
make install
构建共享库
make clean
./configure --shared
make test
make install
cp zutil.h /usr/local/include
cp zutil.c /usr/local/include
ok
3.

'GLIBCXX_3.4.21' not found

此类问题是因为gcc的版本问题，但是centos通过yum安装的最新版本最高也只能到GLIBCXX_3.4.19,如果你的系统是 ubuntu那就比较好解决，但是如果是centos就比较麻烦，这次以centos7为例简述我的解决过程。
首先检查动态库
strings /usr/lib64/libstdc++.so.6 | grep GLIBC
centos明显没有相应的动态库，在服务器内部看能不能找到相关的文件
find / -name "libstdc++.so*"

不过幸运的是刚好我以前在这个服务器里面玩过docker，居然在 docker的目录中找到了相应的libstdc++.so.6.0.25，这肯定是高于运行所需要的GLIBCXX_3.4.21版本了。
所以我就从docker中copy了一个出来，放到了/usr/lib64文件中。
然后就添加软链接
ln -s libstdc++.so.6.0.25 libstdc++.so.6
如果文件已存在就直接强制覆盖
ln -sf libstdc++.so.6.0.25 libstdc++.so.6

###四.优化
虽然启动了服务，但是我们需要的是服务能挂起而不需要我们长期停留在窗口。这里推荐使用screen。
yum install screen
安装完成之后，执行
screen
然后就可以在新的窗口中执行bedrock-server运行程序，运行完成直接关闭窗口就行

五. 总结

用ubuntu系统运行 bedrock-server是最好的解决办法，如果只有centos，那推荐用docker启动ubuntu镜像来运行。强行用centos系统来运行bedrock-server不是不行，只是很麻烦，最后还是缺少一个文件，不得不从ubuntu系统中copy一个过来。
